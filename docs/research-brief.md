# Winning brief (verified research, June 2026)

Every claim below survived 3-vote adversarial verification against primary sources. This brief is binding on all build and review agents.

## What wins, validated

1. **"Feels like a product, not a hackathon project."** The GitLab AI Hackathon 2026 grand prize winner (LORE) was a multi-agent system with a router. A judge from Anthropic praised it in exactly those words. Hive's architecture matches the winning archetype. (Source: GitLab winners blog.)
2. **Production polish is a documented differentiator.** The same winner shipped 43 unit tests, called "extraordinarily rare for the format." We ship real unit tests on swarm logic (orchestrator tick, claim safety, event reducer).
3. **Judges do not reward complexity for its own sake.** Organizer guidance (lablab.ai Internet of Agents): a real working demo solving a clear problem, clean readable code, usable interfaces, reusable value. Hive must be framed as a product: "delegate real work to a transparent agent team," not a tech flex.
4. **The demo video is the first artifact judges see** and often dominates first impressions before code is opened. (Devpost official guidance, 3-0 verified.)
5. **Screencast with narration beats a marketing video.** Devpost explicitly recommends screen capture with voice over "snazzy marketing videos." Cinematic visuals of the real product running: yes. Stylized promo that obscures function: no.
6. **Say what it does, and the InsForge fit, in the first seconds** of the video, like a movie trailer opening. (Devpost, 3-0 verified.)
7. **Judging dimensions to assume** (from comparable AI hackathon): technical execution, design quality, potential impact, idea quality. Cover all four explicitly in README and video.

## Binding implications for Hive

- Frame: HIVE is a product. Tagline: "Delegate a goal. Watch a team of AI agents plan, execute, review, and deliver, live." The 3D scene is not decoration; it is the product's transparency feature (you can see your agents think).
- Demo video (under 3 min): narrated screencast. Structure: (0:00) name + one-line value + "runs entirely on InsForge"; (0:20) type a goal, swarm wakes; (1:30) cut to InsForge dashboard: tables, functions, vectors, realtime filling up; (2:20) artifact download + hosted-on-InsForge URL reveal; (2:50) close.
- Quality gates: zero console errors, no dead states, unit tests passing, clean README with architecture diagram.
- Scope discipline: cut features before cutting polish. A smaller flawless surface beats a bigger rough one.
